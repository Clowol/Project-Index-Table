## 1. 速查表（选型用）

| 协议 | 类型 | 最大速率 | 典型距离 | 引脚数 | 特点 | 典型场景 |
|------|------|----------|----------|--------|------|----------|
| UART | 异步串行 | 1Mbps+ | 1-5m | 2 (TX,RX) | 点对点，简单 | 调试日志、GPS、蓝牙模块 |
| I2C | 同步串行 | 400kbps~3.4Mbps | 0.5-2m | 2 (SDA,SCL) | 多设备，带地址 | 传感器（MPU6050、BME280） |
| SPI | 同步串行 | 10-50Mbps | 0.5-1m | 4+ (MISO,MOSI,SCK,CS) | 高速，全双工 | 显示器、SD卡、高速ADC |
| CAN | 差分串行 | 1Mbps | 40-500m | 2 (CANH,CANL) | 多主，抗干扰，仲裁 | 机械臂关节、车载网络 |
| RS-485 | 差分串行 | 10Mbps | 1200m | 2 (A,B) | 多点，抗共模干扰 | 工业总线、远距离传感器 |
| Ethernet | 网络 | 10/100/1000Mbps | 100m | 4/8 | 高吞吐，复杂协议栈 | 上位机通信、ROS |
| WiFi | 无线 | 11-数百Mbps | 30-100m | 无 | 高带宽，功耗较高 | 远程控制、视频传输 |
| BLE | 无线 | 1-2Mbps | 10-30m | 无 | 低功耗，小数据包 | 手机遥控、传感器节点 |
| USB | 有线 | 12Mbps~40Gbps | 3-5m | 4 | 即插即用，高带宽 | 调试烧录、摄像头、虚拟串口 |

## 2. 各协议一句话总结

| 协议 | 一句话总结 |
|------|-------------|
| **UART** | 最简单、最通用的点对点异步通信，适合调试和低速外设。 |
| **I2C** | 两根线挂一堆传感器，地址寻址，适合板内短距离低速总线。 |
| **SPI** | 高速全双工，四线制，适合高吞吐量设备（屏幕、SD卡）。 |
| **CAN** | 差分信号，抗干扰，多主仲裁，适合实时控制和车载/机器人总线。 |
| **RS-485** | 差分长距离，多节点，工业现场首选，常配 Modbus 协议。 |
| **Ethernet** | 高带宽，TCP/IP 协议栈，适合与 PC/云通信。 |
| **WiFi** | 无线高带宽，适合远程监控和互联网接入，但功耗高。 |
| **BLE** | 低功耗无线，适合手机交互和电池供电设备。 |
| **USB** | 通用高速有线，即插即用，适合调试和批量数据传输。 |

## 3. 选型决策树（按场景）

### 3.1 我需要通信什么类型的数据？
- **控制指令（实时性要求高）**：CAN、EtherCAT（后续补充）
- **传感器数据（周期性、小数据量）**：I2C、SPI、UART
- **调试日志**：UART
- **图像/大文件**：SPI（SD卡）、USB、Ethernet、WiFi

### 3.2 通信距离？
- **板内（<10cm）**：I2C、SPI
- **机箱内（<1m）**：UART、CAN、SPI（降速）
- **设备间（1-10m）**：CAN、RS-485、WiFi
- **远距离（>10m）**：RS-485、Ethernet、LoRa（后续补充）

### 3.3 节点数量？
- **点对点**：UART、SPI、USB
- **多点（<10）**：I2C、CAN
- **多点（>10）**：RS-485、CAN（扩展）

### 3.4 环境干扰？
- **电机、变频器强干扰**：CAN、RS-485（差分信号）
- **普通环境**：I2C、UART、SPI

## 4. 机器人系统中的通信架构示例
以一个典型的机械臂系统为例：
- **主控制器（STM32）**：通过 CAN 总线与各关节
- **关节控制器（STM32）**：通过 CAN 接收指令，控制电机驱动器
- **传感器（IMU、力传感器）**：通过 I2C/SPI 接入关节控制器
- **上位机通信**：通过 Ethernet 或 WiFi 与主控制器通信，实现远程监控和调试
- **调试接口**：通过 UART 输出日志，方便开发调试
